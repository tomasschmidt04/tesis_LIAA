# Gaze-Supervised Transformers

This repository contains code for gaze-supervised fine-tuning of pre-trained language models, together with an extended Spanish pipeline built around BETO and measured scanpaths.

The original core of the repository implements the ACL 2024 idea of augmenting a standard Transformer classifier with a scanpath-aware auxiliary branch during training. The auxiliary branch is removed at inference time, so downstream usage remains compatible with standard Transformer workflows. In addition to the original BERT and RoBERTa GLUE pipelines, the repository now includes a BETO-based extension for Spanish masked-language-model training on measured scanpaths and downstream transfer to Spanish classification benchmarks.

## Research Objective

The project studies whether human reading behavior can improve Transformer representations. Concretely, it introduces gaze supervision as an auxiliary signal during training:

- A Transformer encodes the text in standard token order.
- A scanpath module produces or consumes an ordered sequence of fixations.
- The hidden states visited by that scanpath are re-encoded with a GRU.
- An auxiliary loss encourages the model to align internal representations with human reading behavior.

The main hypothesis is that gaze-informed supervision can improve downstream generalization while preserving standard Transformer inference.

## Architecture

### 1. Original sequence-classification architecture

The original ACL-style sequence-classification pipeline is implemented in:

- [Gazesup_bert_model.py](./Gazesup_bert_model.py)
- [Gazesup_roberta_model.py](./Gazesup_roberta_model.py)

Its training-time architecture is:

1. Text is encoded by a standard Transformer backbone such as BERT or RoBERTa.
2. A scanpath generator produces an ordered sequence of word-level fixations.
3. Word-level fixations are converted into token-level positions.
4. The corresponding token hidden states are selected in scanpath order.
5. A GRU encodes the scanpath-ordered hidden-state sequence.
6. The model combines the standard text loss with a gaze-supervised auxiliary loss weighted by `augweight`.

In the original GLUE scripts, the auxiliary branch is used only during training. At inference time, the workflow remains a standard sequence-classification pipeline.

### 2. Scanpath sources

The repository supports two scanpath sources:

- `eyettention`: scanpaths are generated by the pretrained Eyettention module embedded in the gaze-supervised BERT and RoBERTa models.
- `measured`: scanpaths are loaded from an aligned external dataset and injected directly into the model.

The measured-scanpath utilities live in [measured_scanpath_utils.py](./measured_scanpath_utils.py).

### 3. BETO measured-scanpath MLM extension

The Spanish extension introduces an MLM-based training pipeline built around BETO:

- [Gazesup_bert_mlm_model.py](./Gazesup_bert_mlm_model.py)
- [Gazesup_bert_combined_mlm_model.py](./Gazesup_bert_combined_mlm_model.py)
- [train_mlm_scanpath_step5.py](./train_mlm_scanpath_step5.py)
- [train_mlm_combined_step6.py](./train_mlm_combined_step6.py)
- [train_mlm_combined_step7.py](./train_mlm_combined_step7.py)

In this variant:

1. BETO encodes the original token sequence.
2. Measured word-level scanpaths are expanded to token-level fixation sequences.
3. The selected token hidden states are fed to a GRU in scanpath order.
4. The GRU output remains at scanpath length `S`.
5. An auxiliary MLM head predicts directly over the scanpath-level sequence.
6. For the combined model, the total loss is:

```text
total_loss = main_mlm_loss + aux_weight * scanpath_mlm_loss
```

This implementation explicitly preserves repeated fixations: if a masked token is revisited multiple times in the scanpath, it contributes multiple times to the auxiliary MLM loss.

### 4. Downstream usage after pretraining

After MLM training, the reusable Transformer backbone can be exported with:

- [export_and_verify_backbone_step8.py](./export_and_verify_backbone_step8.py)

Spanish downstream evaluation is then run with:

- [run_spanish_downstream_from_l7b_step9.py](./run_spanish_downstream_from_l7b_step9.py)
- [run_spanish_baseline_beto_step9b.py](./run_spanish_baseline_beto_step9b.py)
- [train_spanish_downstream_baseline.py](./train_spanish_downstream_baseline.py)

As in the original paper setup, downstream classification uses only the main Transformer backbone plus a new task head. No GRU, no scanpath branch, and no auxiliary loss are used at inference time.

## Installation

### 1. Clone the repository

```bash
git clone https://github.com/aeye-lab/ACL-GazeSupervisedLM
cd ACL-GazeSupervisedLM
```

### 2. Create an environment and install dependencies

On Windows PowerShell:

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

On Linux or macOS:

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

### 3. Feature-normalization files

The repository already includes:

- `bert_feature_norm_celer.pickle`
- `roberta_feature_norm_celer.pickle`

If you need to re-download them, use:

```bash
wget https://github.com/aeye-lab/ACL-GazeSupervisedLM/releases/download/v1.0/bert_feature_norm_celer.pickle
wget https://github.com/aeye-lab/ACL-GazeSupervisedLM/releases/download/v1.0/roberta_feature_norm_celer.pickle
```

### 4. Notes on execution environment

- The original GLUE launchers are Bash scripts. On Windows, run them from Git Bash or translate them into direct Python invocations.
- Several provided launchers hard-code `CUDA_VISIBLE_DEVICES`. Adjust the device index before reproducing experiments on a different machine.
- The code depends on Hugging Face `transformers`, `datasets`, and `evaluate`, and downloads benchmark datasets automatically when possible.

## Data and Scanpaths

### GLUE

The original sequence-classification experiments download GLUE via the Hugging Face `datasets` library.

### Measured scanpaths

The measured-scanpath pipeline expects an aligned file in `json`, `jsonl`, or `csv` format. The canonical loader is implemented in [measured_scanpath_utils.py](./measured_scanpath_utils.py).

At minimum, the measured file should contain:

- `text`: the exact sentence consumed by the tokenizer
- `word_id`: a 1-based lexical fixation sequence

Internally, the helper:

- tokenizes the sentence at the word level,
- computes token-to-word alignment,
- injects start/end sentinels,
- converts the lexical fixation sequence into token-level scanpath positions.

The current BETO experiments were run with aligned Spanish reading data such as:

```text
..\reading-et\aligned_output\aligned_scanpaths.jsonl
```

That aligned dataset is not redistributed in this repository.

### Spanish downstream tasks

The Spanish downstream scripts use:

- `xnli_es` from `datasets.load_dataset("xnli", "es")`
- `intertass2020` from the Hugging Face `iberbench/iberbench_all` release, downloaded as a parquet file and cached locally

## Reproducing the BETO Spanish Pipeline

The BETO workflow is an extension of the original repository and is organized as a multi-stage experimental pipeline.

### Stage 1. Train the combined MLM model with measured scanpaths

This is the current long-running BETO pretraining command documented in `paso_7b`:

```powershell
.\.venv\Scripts\python.exe train_mlm_combined_step7.py --measured_scanpath_file "..\reading-et\aligned_output\aligned_scanpaths.jsonl" --model_name_or_path dccuchile/bert-base-spanish-wwm-cased --measured_text_field text --measured_word_id_field word_id --split train --output_dir Pasos/paso_7b --max_seq_length 128 --max_train_samples 2000 --max_eval_samples 256 --per_device_train_batch_size 4 --per_device_eval_batch_size 4 --num_train_epochs 5 --learning_rate 5e-05 --max_masked_positions 3 --aux_weight 0.1 --save_every_epoch True --seed 13
```

Outputs:

- `Pasos/paso_7b/checkpoint_epoch_*`
- `Pasos/paso_7b/best_checkpoint`
- `Pasos/paso_7b/checkpoint_final`

### Stage 2. Export a clean downstream backbone

```powershell
.\.venv\Scripts\python.exe export_and_verify_backbone_step8.py --source_checkpoint "Pasos/paso_7b/best_checkpoint" --output_dir Pasos/paso_8b --export_dirname export_backbone_hf --downstream_num_labels 2
```

This exports the reusable main Transformer backbone without the scanpath GRU and auxiliary MLM heads.

### Stage 3. Run Spanish downstream transfer from the l7b checkpoint

```powershell
.\.venv\Scripts\python.exe .\run_spanish_downstream_from_l7b_step9.py --l7b_dir .\Pasos\paso_7b --output_dir .\Pasos\paso_9_es_l7b --max_train_samples 500 --max_eval_samples 200 --num_train_epochs 3 --per_device_train_batch_size 8 --per_device_eval_batch_size 8 --max_seq_length 128 --learning_rate 2e-5 --seed 13
```

This wrapper automatically resolves the checkpoint to use inside `Pasos/paso_7b` with the following priority:

1. explicit best checkpoint,
2. latest valid checkpoint,
3. root directory fallback.

It runs exactly two Spanish downstream tasks:

- `xnli_es`
- `intertass2020`

### Stage 4. Run the direct BETO baseline

```powershell
.\.venv\Scripts\python.exe .\run_spanish_baseline_beto_step9b.py --model_name_or_path dccuchile/bert-base-spanish-wwm-cased --output_dir .\Pasos\paso_9b_beto --step9_reference_dir .\Pasos\paso_9_es_l7b --max_train_samples 500 --max_eval_samples 200 --num_train_epochs 3 --per_device_train_batch_size 8 --per_device_eval_batch_size 8 --max_seq_length 128 --learning_rate 2e-5 --seed 13
```

This is the comparison baseline: same downstream tasks, same hyperparameters, same evaluation protocol, but without prior scanpath-aware pretraining.

## Model Variants

The repository currently exposes the following experimental variants.

### Original ACL-style variants

- `train_glue_LM_baseline.py`: standard sequence-classification fine-tuning without gaze supervision
- `train_glue_gazesup_bert_high_resource.py`: BERT with gaze-supervised auxiliary branch
- `train_glue_gazesup_bert_low_resource.py`: low-resource BERT gaze-supervised variant
- `train_glue_gazesup_roberta_high_resource.py`: RoBERTa with gaze-supervised auxiliary branch
- `train_glue_gazesup_roberta_low_resource.py`: low-resource RoBERTa gaze-supervised variant

### BETO and Spanish extensions

- `Gazesup_BERTForMaskedLM`: measured-scanpath auxiliary MLM branch over scanpath length `S`
- `Gazesup_BERTForCombinedMaskedLM`: standard MLM plus scanpath-aware auxiliary MLM
- `run_spanish_downstream_from_l7b_step9.py`: downstream transfer from the trained BETO `l7b` experiment
- `run_spanish_baseline_beto_step9b.py`: direct BETO baseline for fair comparison

## Reproducibility Guidelines

For academic reproducibility, keep the following fixed:

- model identifier
- task set
- training/evaluation split definition
- `max_train_samples`
- `max_eval_samples`
- number of epochs
- batch sizes
- learning rate
- random seed
- checkpoint-selection policy

For the Spanish comparison specifically, `paso_9` and `paso_9b` are designed to be directly comparable:

- same tasks: `xnli_es`, `intertass2020`
- same tokenizer interface per task
- same maximum sequence length
- same sample caps
- same number of epochs
- same batch sizes
- same learning rate
- same seed
- only the initialization changes

This makes the `l7b` vs. direct BETO comparison the cleanest current test of whether the scanpath-aware pretraining stage contributes useful transfer signal.
